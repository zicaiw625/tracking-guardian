# Demo Store 配置指南

本文档说明如何配置 Demo Store 用于 Shopify App Store 审核。

## Demo Store 要求

### 1. 商店类型

- **开发商店** (Development Store)
- 已启用 **Checkout Extensibility**
- 已安装 **Web Pixel Extension**（通过本应用）

### 2. 测试数据准备

#### 订单数据
- 至少 5-10 个测试订单
- 订单金额范围：$10 - $500
- 包含不同币种的订单（如果支持多币种）
- 订单状态：已完成（paid）

#### 追踪配置
- 至少配置 2 个广告平台（如 Google GA4 和 Meta）
- 配置服务器端追踪（CAPI）
- 配置客户端追踪（Web Pixel）

#### 测试场景
- 标准购买流程
- 多商品购买
- 退款流程（可选）
- 订阅订单（可选）

## Demo Store 配置步骤

### 步骤 1: 创建开发商店

1. 登录 Shopify Partners Dashboard
2. 创建新的开发商店
3. 选择 "Development store" 类型
4. 启用 "Checkout Extensibility" 功能

### 步骤 2: 安装应用

1. 在开发商店中安装本应用
2. 完成 OAuth 授权流程
3. 确认应用已正确安装

### 步骤 3: 配置测试追踪

1. **配置 Google GA4**:
   - 进入应用设置页面
   - 选择 "像素迁移" 功能
   - 配置 GA4 Measurement ID 和 API Secret
   - 启用服务器端追踪

2. **配置 Meta Pixel**:
   - 配置 Pixel ID 和 Access Token
   - 启用服务器端追踪（CAPI）

3. **验证配置**:
   - 运行验收向导
   - 完成测试订单
   - 验证事件发送成功

### 步骤 4: 创建测试订单

1. 使用测试支付方式创建订单
2. 完成至少 5 个订单
3. 等待事件处理完成（通常 1-2 分钟）

### 步骤 5: 验证功能

确保以下功能可以正常演示：

- ✅ Audit 扫描功能
- ✅ 像素迁移向导
- ✅ 事件对账功能
- ✅ 监控面板
- ✅ 告警通知（可选）

## 常见问题

### Q: Demo Store 需要多少测试数据？

A: 建议至少 5-10 个测试订单，涵盖不同的场景（单商品、多商品、不同金额等）。

### Q: 需要配置真实的广告平台账号吗？

A: 建议配置真实的测试账号，但可以使用测试 Pixel ID 和 API Key。确保不会产生实际费用。

### Q: 如何验证事件发送成功？

A: 使用应用内的"验收向导"功能，它会自动验证事件是否成功发送到配置的平台。

## 注意事项

1. **数据隐私**: Demo Store 中不要使用真实的客户数据
2. **API 限制**: 注意测试平台的 API 调用限制
3. **费用控制**: 使用测试账号避免产生实际费用
4. **定期更新**: 定期更新测试数据以保持 Demo Store 的可用性
